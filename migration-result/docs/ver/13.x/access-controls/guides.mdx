Teleport gives you fine-grained control over who can access resources in your
infrastructure as well as how they can access those resources. Once you have
deployed a Teleport cluster, configure access controls to achieve the right
security policies for your organization.

- [Dual Authorization](/docs/ver/13.x/access-controls/guides/dual-authz): Protect access to critical resources with dual authorization.
- [Role Templates](/docs/ver/13.x/access-controls/guides/role-templates): Set up Dynamic Access Policies with Role Templates.
- [Impersonating Teleport Users](/docs/ver/13.x/access-controls/guides/impersonation): Create certificates for CI/CD with impersonation.
- [Passwordless](/docs/ver/13.x/access-controls/guides/passwordless): Use passwordless authentication.
- [Multi-Factor: WebAuthn](/docs/ver/13.x/access-controls/guides/webauthn): Add multi-factor authentication through WebAuthn.
- [Per-Session MFA](/docs/ver/13.x/access-controls/guides/per-session-mfa): Per-session multi-mactor authentication.
- [Locking](/docs/ver/13.x/access-controls/guides/locking): Lock access to active user sessions or hosts.
- [Moderated Sessions](/docs/ver/13.x/access-controls/guides/moderated-sessions): Require session auditors and allow fine-grained live session access.
- [Hardware Key Support](/docs/ver/13.x/access-controls/guides/hardware-key-support): Enforce the use of hardware-based private keys.
- [Device Trust (Preview)](/docs/ver/13.x/access-controls/device-trust/guide): Register and enforce trusted devices.
- [Headless WebAuthn (Preview)](/docs/ver/13.x/access-controls/guides/headless): Login with Webauthn from a remote device.
- [IP Pinning (Preview)](/docs/ver/13.x/access-controls/guides/ip-pinning): Pin a user's certificates to a login IP address.
