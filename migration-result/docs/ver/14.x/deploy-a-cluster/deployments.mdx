These guides show you how to set up a full self-hosted Teleport deployment on
the platform of your choice.

- [AWS High Availability Deployment with Terraform](/docs/ver/14.x/deploy-a-cluster/deployments/aws-ha-autoscale-cluster-terraform): Deploy HA Teleport with
  Terraform on AWS.
- [AWS Single-Instance Deployment with Terraform](/docs/ver/14.x/deploy-a-cluster/deployments/aws-starter-cluster-terraform): Deploy Teleport on a single instance with
  Terraform on AWS.
- [AWS Multi-Region Proxy
  Deployment](/docs/ver/14.x/deploy-a-cluster/deployments/aws-gslb-proxy-peering-ha-deployment): Deploy HA
  Teleport with Proxy Service instances in multiple regions for low-latency
  access.
- [GCP](/docs/ver/14.x/deploy-a-cluster/deployments/gcp): Deploy HA Teleport on GCP.
- [IBM Cloud](/docs/ver/14.x/deploy-a-cluster/deployments/ibm): Deploy HA Teleport on IBM cloud.
