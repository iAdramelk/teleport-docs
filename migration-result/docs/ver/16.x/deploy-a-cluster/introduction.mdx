These guides show you how to run a self-hosted Teleport Enterprise or Teleport
Community Edition cluster in production.

Read our [High Availability Guide](/docs/ver/16.x/deploy-a-cluster/high-availability) for the general
principles behind deploying a scalable, fault-tolerant Teleport cluster. Once
you understand what is required to run Teleport in production, choose your
compute platform and cloud provider:

## Deploy on Kubernetes using Helm

You can deploy a complete, high-availability Teleport cluster on Kubernetes
using our `teleport-cluster` Helm chart. These guides show you how to deploy the
`teleport-cluster` Helm chart on your cloud:

- [Google Cloud](/docs/ver/16.x/deploy-a-cluster/helm-deployments/gcp)
- [AWS](/docs/ver/16.x/deploy-a-cluster/helm-deployments/aws)
- [Kubernetes with SSO](/docs/ver/16.x/deploy-a-cluster/helm-deployments/kubernetes-cluster)
- [Custom Kubernetes Deployment](/docs/ver/16.x/deploy-a-cluster/helm-deployments/custom)

## Deploy using virtual machines

You can run a production cluster of Teleport instances using virtual machines.
In these guides, we will show you how to deploy a high-availability, VM-based
Teleport cluster on your cloud:

- [AWS High Availability Deployment with Terraform](/docs/ver/16.x/deploy-a-cluster/deployments/aws-ha-autoscale-cluster-terraform)
- [AWS Single-Instance Deployment with Terraform](/docs/ver/16.x/deploy-a-cluster/deployments/aws-starter-cluster-terraform)
- [AWS Multi-Region Proxy Deployment](/docs/ver/16.x/deploy-a-cluster/deployments/aws-gslb-proxy-peering-ha-deployment)
- [Google Cloud](/docs/ver/16.x/deploy-a-cluster/deployments/gcp)
- [IBM Cloud](/docs/ver/16.x/deploy-a-cluster/deployments/ibm)
