The guides in this section show you the fundamentals of setting up and running a
Teleport cluster. You will learn how to run the `teleport` daemon, manage users
and resources, and troubleshoot any issues that arise.

If you already understand how to set up a Teleport cluster, consult the
[Operations](/management/operations) section so you can start conducting periodic
cluster maintenance tasks.

## Run Teleport

- [Teleport Daemon](/management/admin/daemon): Set up Teleport as a daemon on Linux with systemd.
- [Upgrade the Teleport Binary](/management/admin/upgrading-the-teleport-binary): Upgrade the `teleport` binary without losing connections.
- [Run Teleport with Self-Signed Certificates](/management/admin/self-signed-certs): Set up Teleport in a local
  environment without configuring TLS certificates.

## Manage users and resources

- [Trusted Clusters](/management/admin/trustedclusters): Connect multiple Teleport clusters using trusted clusters.
- [Labels](/management/admin/labels): Manage resource metadata with labels.
- [Local Users](/management/admin/users): Manage local user accounts.

## Troubleshoot issues

- [Troubleshooting](/management/admin/troubleshooting): Collect metrics and diagnostic information from Teleport.
- [Uninstall Teleport](/management/admin/uninstall-teleport): Uninstall Teleport from your system.
