These guides cover how to configure a deployed Machine ID to produce credentials
that can be used for machine to machine access to different Teleport resources.

It is a pre-requisite of these guides that Machine ID has been configured for
your platform, see the [Deploy Machine ID](/machine-id/deployment) guides for information
on how to do so.

## Resource Access

- [Server Access](/machine-id/access-guides/ssh): How to use Machine ID to access servers via SSH.
- [Kubernetes Access](/machine-id/access-guides/kubernetes): How to use Machine ID to access Kubernetes clusters.
- [Database Access](/machine-id/access-guides/databases): How to use Machine ID to access Database servers.
- [Application Access](/machine-id/access-guides/applications): How to use Machine ID to access Applications.

## Specific Tools

- [tctl](/machine-id/access-guides/tctl): How to use Machine ID with `tctl` to manage your Teleport configuration.
- [Teleport Terraform](/machine-id/access-guides/terraform): How to use Machine ID with the Teleport Terraform provider to manage your Teleport configuration as IaC.
- [Ansible](/machine-id/access-guides/ansible): How to use Machine ID with Ansible.
